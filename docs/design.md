# 系统设计文档

## 1. 项目目标
构建一个面向用户行为日志的风险分析系统，对用户访问行为进行统计、特征提取和异常检测。

## 2. 模块划分

### 2.1 数据加载模块
负责读取 CSV 日志文件，并完成基础清洗。

### 2.2 特征工程模块
从日志中提取用户总操作次数、点击次数、访问页面数、活跃时长、操作速度等特征。

### 2.3 风险检测模块
基于规则对用户行为进行评分，并划分高风险、中风险、低风险等级。

### 2.4 报告输出模块
输出用户行为统计表和风险识别结果。

## 3. 可扩展方向
1. 引入 sklearn 异常检测模型
2. 接入数据库进行真实 SQL 查询
3. 增加 Web 可视化页面
4. 接入 Java 服务层实现接口化封装
